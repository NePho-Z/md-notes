# 串口部分

- 和电脑通信要下载串口驱动与串口助手，还有ttl转usb通信模块

## 1.直接输入输出模式(轮询模式）必须等待发送结束，只能接收定长数据

- `HAL_UART_Transmit(UART_HandleTypeDef *huart, const uint8_t *pData, uint16_t Size, uint32_t Timeout)`
- `HAL_UART_Receive(UART_HandleTypeDef *huart, uint8_t *pData, uint16_t Size, uint32_t Timeout)`
- 串口指针，数据指针，数据长度，最大时间（无限等待HAL_MAX_DELAY)
- 串口原理数据先到tdr再到发送移位寄存器变成二进制输出；
- 接收模式同理，cpu会一直查询接受数据寄存器中是否有新数据可读，一旦检测到，就会放入变量，此时cpu堵塞，无法执行其他任务

## 2.中断模式（解决堵塞问题）

- 检测发送数据寄存器是否为空，如果为空，触发中断
- 配置方法sys core -- NVIC -- USARTx global interrupt
- `HAL_UART_Transmit_IT(UART_HandleTypeDef *huart, const uint8_t *pData, uint16_t Size)`
- `HAL_UART_Receive_IT(UART_HandleTypeDef *huart, uint8_t *pData, uint16_t Size)`
  - 接收数据时，由于不堵塞程序（指先去处理其它代码），可能数据还没接收完就开启了下一次接收数据，故不能直接处理，要使用中断处理函数

### 中断处理函数如图，只有一个函数，对应很多事件

- callback回调函数处理具体的事件，回调函数：当有什么事情发生时，就会调用这个函数，故可以精确到一个事件
- `__weak void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)这个函数会在接收完成时进行（既是串口中断，又是dma传输完成中断）（适用于定长数据）`

## 3.dma模式（用来帮助cpu搬运数据）（内存变量和发送数据寄存器）

- 是否地址自增。寄存器不用，内存变量要

### 接收不定长数据（使用串口空闲中断）

- `HAL_UARTEx_ReceiveToIdle_DMA(UART_HandleTypeDef *huart, uint8_t *pData, uint16_t Size)`
- `回调函数__weak void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)习惯：判断谁触发了中断函数`

### 接收定长数据

- `DMA：串口DMA配置一切默认发送接收把_IT改为_DMA，关闭DMA传输过半中断HAL_DMA_DISABLE_IT(&hdma_usartx_rx,DMA_IT_HT)`

## 接收数据时的奇怪bug

- 你的接收缓冲区放在自定义段，但链接脚本没有这个段
- 缓冲区在 [.RAM_D2_Array]，见 [main.c:51](http://main.c:51)。
- 但链接脚本里没有 .RAM_D2_Array 输出段定义，见 [STM32H743XX_FLASH.ld](http://STM32H743XX_FLASH.ld)。
- 而当前数据段和 bss 都被放到 DTCMRAM，见 [STM32H743XX_FLASH.ld:168](http://STM32H743XX_FLASH.ld:168)、[STM32H743XX_FLASH.ld:216](http://STM32H743XX_FLASH.ld:216)、[STM32H743XX_FLASH.ld:226](http://STM32H743XX_FLASH.ld:226)。
- 在 H743 上，DMA1 访问 DTCM 是有问题的，这正好会造成“CPU 阻塞收能行，DMA 收不行”。
- 之后dma收不到数据可能是这个让ai改一下即可
