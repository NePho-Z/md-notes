### 浏览器走代理，git不一定走代理，之后其他软件也可以通过这种方式大大加快速度

- `git config --global http.proxy ... 的意思是：把 Git 的“HTTP 代理”设置写到你的全局 Git 配置里，让 Git 以后通过这个代理去访问走 HTTP/HTTPS 的仓库地址（例如 https://github.com/...）。`
- 拆开解释：
- `git config：修改/读取 Git 配置`
- --global：写入当前用户的全局配置（通常在 C:\Users\<你>\.gitconfig）。以后在任何仓库里都生效。
- http.proxy：Git 用于 HTTP 请求的代理设置项（虽然叫 http.proxy，但访问 https://... 通常也会受它/https.proxy 影响）
- 你在后面接的值（比如 http://127.0.0.1:7890）：就是代理地址（本机 + 端口）
- 例如你用 FClash 的 mixed-port: 7890，常见就这样配：
- PowerShell
- `git config --global http.proxy http://127.0.0.1:7890git config --global https.proxy http://127.0.0.1:7890`
- 如何查看你现在设置成什么了
- PowerShell
- `git config --global --get http.proxygit config --global --get https.proxy`
- 如何取消（恢复不走代理）
- PowerShell
- `git config --global --unset http.proxygit config --global --unset https.proxy`
- 补充：你前面 cargo run 拉依赖超时，很多时候就是因为 Git/Cargo 没走代理。把 Git 的代理指到 127.0.0.1:7890，通常能明显改善。
